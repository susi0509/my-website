##   1）OSLC的核心思想——Linked Data

关联数据是由Tim Burners-Lee首先提出的结构化数据，用于发布数据。关联数据被视为灵活、标准化的开源基础方法，为共享和集成不同工具产生的异构数据（企业内部或者外部的“信息孤岛”）提供支持。这种格式有助于在数据连接中实现更好的互连，并将数据转换为简单，易于集成和结构化的方式。

关联数据术语利用指定实现的方法，以将数据与其资源互连，例如Web架构概念使用http链接和URI从一个位置转移到另一个位置。使用RDF作为其基本数据模型并使用SPARQL作为其查询语言来指定关联数据。考虑的重要属性是它提供了RESTful服务中的结构源，并且可以用作机器和人类都能理解的其他工具的元数据。关联数据的四个原则是，

-  使用URI作为事物的名称

-  使用HTTP URI，以便人们可以查找这些名称。

-  当有人查找URI时，使用标准（RDF *，SPARQL）提供有用的信息。

- 包含指向其他URI的链接，这样他们就能发现更多的东西。

Linked Data可以概述为：资源都通过HTTP URI进行标识，用户通过请求能够获取通过标准形式表述的有用信息，并且允许事物间的链接，使得用户能够发现更多的信息。

##  2）OSLC的主要集成技术

OSLC提供了两种集成工具的主要技术—— “通过HTTP链接数据”和“通过HTML用户界面链接数据”。这两种技术都建立在OSLC中HTTP和RDF基础之上。

-  通过HTTP链接数据

OSLC指定了用于创建、检索、更新和删除（CRUD）生命周期数据的通用工具协议，这些数据基于互联网标准，比如HTTP和使用Linked数据模型的RDF。任何工具或其他程序客户端都可以使用此协议和任何其他实现规范的工具进行通信。通过将一个资源的HTTP URL嵌入另一个资源的表示中来实现链接。

-  通过HTML用户界面链接数据

OSLC指定一种协议，允许工具或其他客户端显示另一个工具的Web用户界面的片段，允许人类用户链接到另一个工具中的新资源或现有资源，或者查看另一个工具中的资源的信息预览。这使工具或其他客户端可以在整合信息和流程步骤时可以利用现有的用户界面和其他工具中的业务逻辑。在某些情况下，这比实现新的用户界面更高效并提供更多的用户功能，然后通过HTTP的CRUD协议进行集成。
